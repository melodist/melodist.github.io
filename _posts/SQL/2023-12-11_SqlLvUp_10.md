# 인덱스 사용
## 인덱스와 B-tree
### 만능형: B-tree
- 데이터를 트리 구조로 저장하는 형태
- 대부분의 데이터베이스에서는 트리의 리프 노드에만 키값을 저장하는 B+tree 사용
- 루트와 리프의 거리를 가능한 일정하게 유지하므로 검색 성능이 안정적
### 기타 인덱스
- 비트맵 인덱스
  - 데이터를 비트 플래그로 변환
  - 카디널리티가 낮은 필드에 유효
  - 갱신할 때 오버헤드가 크므로 BI/DWH 용도로 사용
- 해시 인덱스
  - 키를 해시 분산해서 등가 검색을 고속으로 실행
  - 등가 검색 외에는 효과가 거의 없고 범위 검색을 할 수 없음
  - PostgreSQL, Oracle 등 일부 DB만 지원
